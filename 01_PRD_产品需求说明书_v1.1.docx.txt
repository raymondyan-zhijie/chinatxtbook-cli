1. 产品概述
ChinaTextbook v1.1 是面向 Windows 与 Ubuntu 用户的终端教材浏览、按书下载、分卷合并和完整性核验工具。v1.1 将 v1.0 的单文件脚本重构为模块化应用，并使用 Textual 提供键盘优先、可视化、可恢复的终端用户界面。
核心产品变化：仓库固定为 GitHub 官方源；选择颗粒度从目录提升到具体教材；界面升级为三栏 TUI；下载、合并、日志、更新、详情与帮助均在应用内完成。
1.1 产品目标
用户可以通过“学段 → 科目 → 年级/册次 → 教材”快速定位并选择一本或多本教材。
仅下载用户选定教材所需的源文件，不再默认下载整个年级或科目目录。
耗时任务在后台运行，界面保持响应并可安全取消。
下载、分卷恢复、合并、SHA256 校验、状态恢复和更新过程可视、可诊断。
Windows 普通用户可使用 EXE；Ubuntu 用户可通过 wheel/pipx 安装后运行 ctb。

2. 范围与边界
2.1 本版本包含
固定 GitHub 官方仓库 TapXWorld/ChinaTextbook。
Git 树构建的书目索引和书目级 sparse-checkout。
Textual 三栏主界面、搜索、已选教材、下载任务、日志、更新、帮助窗口。
软件更新提示与教材仓库更新提示。
原子合并、SHA256 核验、fail-closed、断点恢复、单实例锁和日志脱敏。
Windows EXE、Python wheel、源码包三种发布物。
2.2 本版本不包含
不提供 Gitee 或其他自动镜像测速与切换。
不自动破解、绕过访问控制或下载非公开资源。
不承诺从 PDF 内容自动推断出版社、ISBN 或教材版本。
不在 v1.1 内实现应用自身的无提示自动升级；仅提示并引导用户查看发布页。
不提供图形桌面 GUI；本版本为终端应用。

4. 产品原则
安全优先
出现清单不确定、分卷结构异常、历史记录冲突时拒绝覆盖，宁可要求人工核对。
书目是用户心智模型
界面展示一本教材，不向普通用户暴露 .pdf.1、.pdf.2 等分卷实现细节。
GitHub 单一事实源
不做镜像测速，不自动切换到无法确认同步状态的仓库。
键盘优先
全部主要功能均可使用方向键、Space、Enter、Esc 和功能键完成。
如实呈现
Git 下载进度若为估算值，必须明确标注“估算”。

6. 非功能需求
类别
要求
兼容性
Windows 10/11 PowerShell、CMD、Windows Terminal；Ubuntu 22.04/24.04 常见终端。
性能
10,000 本以内书目索引首次构建目标小于 5 秒（不含网络）；搜索响应目标小于 200ms。

终端宽度 100 列以上显示三栏；80-99 列自动压缩；低于 80 列显示单栏兼容模式。

任何正式 PDF 写入均使用临时文件、flush、fsync 和原子替换。

可维护性
模块职责清晰、类型标注、核心逻辑与 UI 分离、测试覆盖不低于 80%。
可访问性
颜色不是唯一状态信号；同时使用文字和符号。
7. 状态与数据迁移
新状态必须区分用户选择与 Git 实际文件路径：
字段
含义
selected_books
逻辑教材 ID 列表，例如 小学/数学/一年级/教材.pdf
selected_files
该批教材展开后的实际 Git 文件路径
selection_fingerprint
selected_files 排序后计算的 SHA256
groups
每个 PDF 组的状态、大小、SHA256、历史 parts、stale 标记
tasks
可恢复后台任务的阶段和错误信息
catalog_cache
与 HEAD commit 绑定的书目索引缓存
迁移规则：从 v1.0 升级时保留 groups 和安全哈希记录；清空旧 selected_paths、checkout_done 和目录级缓存；首次启动明确提示用户重新选书。
8. 错误处理与安全约束
Git 树读取失败：停止索引、扫描和合并，不回退到工作区猜测。
单卷分卷组：默认拒绝自动合并。
历史 parts 比新清单多：视为上游删卷或结构异常，保留现有 PDF 并报错。
分卷重复、缺中间卷、大小不一致、哈希不一致：不得删除源分卷。
应用退出：后台任务进入安全取消流程，状态原子保存。
工作区 origin 不是官方 GitHub：拒绝继续，不自动强制重置。

10. 验收标准
用户可从学段进入科目，再进入年级/册次并选择具体教材。
选择一本分卷教材时，仅下载该书对应的全部分卷及必要元数据。
Windows 与 Ubuntu 的方向键、Space、Enter、Esc 和 F1/F2/F5/F6/F8/F9 均符合规范。
下载和合并过程中 UI 不冻结。
应用被关闭后再次运行可继续未完成任务。
上游删卷、清单失败、哈希失败等安全场景均不会覆盖完整 PDF 或误删分卷。
日志和报告不出现明文凭据。
软件更新和教材更新提示不会阻塞主界面。
